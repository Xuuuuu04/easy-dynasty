**问题分析**：

1. **根本原因**：后端缺少 `.env` 配置文件，导致 LLM API Key 为空。
2. **现象解释**：

   * 由于 API Key 为空，后端在调用 LLM 服务时失败（返回 401/403 等）。

   * 但后端代码目前的实现是：即使出错，也返回 HTTP 200 状态码，并将错误信息封装在流（Server-Sent Events）中返回。

   * 前端代码未能正确识别流中的错误信息，误以为是正常的空响应，导致界面显示空白且不报错。

**修复计划**：

1. **后端优化 (backend/app/api/endpoints/tarot.py)**：

   * 在处理请求前，**强制检查 API Key 是否存在**。

   * 如果缺失，直接返回 `500 Internal Server Error`，而不是进入流式响应。这样前端就能立即捕获错误并提示用户。

   * 增加更详细的日志记录，方便排查。

2. **前端增强 (web/hooks/useTarotAnalysis.ts)**：

   * 增强 SSE 解析逻辑：如果流中包含 `error` 字段，立即抛出异常，防止静默失败。

3. **配置引导**：

   * 在后端创建 `.env` 文件模板，并提醒您填入 API Key。

**执行步骤**：

1. 修改后端 `tarot.py`，增加 API Key 检查逻辑。
2. 修改前端 `useTarotAnalysis.ts`，增加流式错误处理。
3. 创建 `backend/.env` 文件。
4. 重启后端服务。
